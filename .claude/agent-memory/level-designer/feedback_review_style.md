User requests adversarial review mode: "Your job is NOT to validate this design — your job is to find problems." Severity labels used: BLOCKING (can't author without this spec), ADVISORY (rough guidance exists but incomplete), COORD-GAP (needs alignment with another department). Domain focus should be level layout, NPC placement, scripted authoring, encounter design.

**Why:** This is a collaborative production environment; GDDs must survive adversarial review before implementation begins. Blind spots caught now prevent rework during sprint.

**How to apply:** When asked to review any GDD, lead with the sharpest domain-specific problems, not with praise. Use the severity labels. Do not soften findings.
